# Code of Conduct

## Introduction

This project is committed to providing a welcoming, respectful, and professional environment for all contributors.

By participating in this project, you agree to uphold the standards outlined in this Code of Conduct.

---

## Our Standards

We expect all participants to:

- Communicate respectfully and professionally
- Be constructive when giving feedback or reviewing code
- Focus on improving the project, not criticizing individuals
- Be open to different viewpoints and experiences
- Show empathy and patience, especially with new contributors


## Examples of Positive Behavior

- Providing helpful and actionable feedback
- Writing clear, respectful pull request reviews
- Asking questions instead of making assumptions
- Helping others understand the codebase or decisions
- Accepting feedback gracefully


## Unacceptable Behavior

The following behaviors are not tolerated:

- Harassment, discrimination, or hate speech
- Personal attacks, insults, or trolling
- Dismissive or hostile communication
- Public or private intimidation
- Any conduct that creates an unwelcoming environment


## Scope

This Code of Conduct applies to all project spaces, including:

- GitHub issues
- Pull requests
- Discussions
- Any communication related to the project


## Enforcement

Project maintainers are responsible for enforcing this Code of Conduct.

They may:

- Remove or edit comments that violate these standards
- Close issues or pull requests that are not constructive
- Restrict or remove contributors who repeatedly violate the rules
